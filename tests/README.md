# اختبارات START LINE الآلية (Playwright)

## ⚠️ حالة مهمة
الاختبارات دي **اتكتبت بناءً على فحص الكود المصدري** (أسماء IDs وclasses الحقيقية في `index.html`)،
**مش بعد تشغيل فعلي في متصفح** — البيئة اللي كتبتها فيها معندهاش وصول لتحميل متصفحات Playwright
(الشبكة مقفولة على npm/pypi/github بس). يعني ممكن يكون فيه سيلكتور واحد أو اتنين محتاجين تعديل بسيط
أول مرة تشغّلهم، خصوصاً في `auth.spec.js` لأنه بيعتمد على افتراض إن مودال الدخول بيتفتح تلقائياً
(الكود بيدعم ده بس مقدرتش أتأكد من التوقيت بالظبط).

## التشغيل لأول مرة

```bash
cd tests
npm install
npx playwright install --with-deps chromium   # تحميل المتصفح (مرة واحدة بس)
```

## تشغيل الموقع محلياً (في تيرمينال منفصل، من جذر المشروع مش من tests/)

⚠️ **مهم بعد تعديل الـ routing (path routing بدل hash):** لازم السيرفر المحلي يدعم SPA
fallback (أي مسار غير موجود كملف حقيقي يرجّع `index.html`)، وإلا `routing.spec.js`
هيفشل في اختبارات الزيارة المباشرة/الـ reload:

```bash
npx serve -s . -l 5500
# فلاج -s (single-page) هو اللي بيعمل SPA fallback محلياً — من غيره هتاخد 404 على /quiz
# أو استخدم vercel dev لمحاكاة إعدادات vercel.json الحقيقية (rewrites) بأدق شكل:
npx vercel dev --listen 5500
```

## تشغيل الاختبارات

```bash
cd tests
npm test
# أو للتشغيل التفاعلي (بيفتح متصفح تقدر تتابع فيه):
npm run test:ui
```

## لو سيلكتور معين فشل
افتح `npm run test:ui`، شغّل الاختبار اللي فشل، وهتشوف بالظبط في أي خطوة وقف —
غالباً هيكون محتاج تحديث بسيط لاسم id أو class لو اتغيّر في الكود بعد كتابة الاختبار ده.

## الملفات
- `specs/smoke.spec.js` — التأكد إن الصفحة بتحمّل من غير أخطاء JS، وإن `app.bundle.min.js`
  الجديد (بعد الدمج) شغال فعلاً، وإن التنقل بين الصفحات (`showPage`) شغال.
- `specs/auth.spec.js` — تدفق تسجيل الدخول الإجباري (مودال بيفتح تلقائياً، validation
  للحقول الفاضية، رسالة خطأ للبيانات الغلط).
- `specs/routing.spec.js` — **الأهم بعد تعديل الروتنج**: التأكد إن URL بيتغيّر لمسار نظيف
  (`/quiz` مش `#quiz`)، إن الزيارة المباشرة والـ reload لمسار زي `/library` بيرجعوا الصفحة
  الصح مش 404، إن زرار الرجوع في المتصفح شغال، وإن أي رابط قديم بصيغة `#quiz` (لو حد عنده
  محفوظ قبل التحديث) لسه بيشتغل ويتحوّل تلقائياً.

## اللي لسه ناقص (اقتراحات للتوسع لاحقاً)
- اختبار تدفق الـ Quiz كامل (بداية → أسئلة → نتيجة)
- اختبار roadmap (فتح مسار، تقدّم خطوة، قفل/فتح خطوات)
- اختبار لوحة الإدارة (admin) — محتاج مستخدم admin تجريبي في قاعدة بيانات اختبار منفصلة، **متستخدمش بيانات production**
- اختبار PWA/offline (Service Worker fallback)
